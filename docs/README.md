## 5. Архитектура приложения «Семейный бюджет»
### 5.1 Описание ключевых компонентов
Проект построен на паттерне MVVM с использованием навигации по страницам (Pages):

Model – бизнес-логика и данные (Transaction, Category, BudgetContext).

View – XAML-страницы (Page), размещаемые внутри главного окна (Frame). Каждая страница имеет свою ViewModel.

ViewModel – содержит команды, наблюдаемые коллекции, свойства для привязки. Управляет навигацией через сервис INavigationService.

Основные классы и страницы:

Pages/HomePage.xaml – главная страница с общей суммой баланса, последними транзакциями.

Pages/TransactionsPage.xaml – список всех доходов/расходов с возможностью добавления, редактирования, удаления.

Pages/CategoriesPage.xaml – управление категориями (создание, редактирование, удаление).

Pages/StatisticsPage.xaml – отображение статистики (диаграмма расходов по категориям, график динамики за месяц).

Вспомогательные компоненты:

BudgetContext – загружает/сохраняет данные в JSON.

RelayCommand – реализация интерфейса ICommand для привязки действий.

NavigationService – централизованное управление переходами между страницами.

5.2 Взаимодействие компонентов
Пользователь нажимает кнопку «Добавить доход» → View вызывает команду AddTransactionCommand в ViewModel → ViewModel открывает диалоговое окно или страницу добавления → после сохранения транзакция попадает в коллекцию → View автоматически обновляет список (благодаря ObservableCollection<T>).

## 6. Техническое руководство для начинающих (создание приложения «Семейный бюджет» на WPF)
### 6.1 Что такое WPF, Page и MVVM
WPF (Windows Presentation Foundation) – технология для создания клиентских приложений Windows с богатым графическим интерфейсом на основе XAML. Page – элемент, предназначенный для навигации (размещается внутри Frame). MVVM (Model-View-ViewModel) – паттерн, отделяющий логику от интерфейса.

### 6.2 Как создать приложение «Семейный бюджет» с нуля
### Шаг 1: Создайте проект WPF в Visual Studio
Выберите шаблон WPF App (.NET). Назовите проект, например, FamilyBudget.

### Шаг 2: Добавьте модели данных
csharp
'''
public class Transaction
{
    public int Id { get; set; }
    public decimal Amount { get; set; }
    public string Category { get; set; }
    public string Type { get; set; } // "Income" или "Expense"
    public DateTime Date { get; set; }
    public string Comment { get; set; }
}
'''
### Шаг 3: Реализуйте контекст данных для работы с JSON
csharp
public class BudgetContext
{
    private readonly string _transactionsFile = "transactions.json";
    public List<Transaction> Transactions { get; set; }

    public void Load()
    {
        if (File.Exists(_transactionsFile))
        {
            string json = File.ReadAllText(_transactionsFile);
            Transactions = JsonSerializer.Deserialize<List<Transaction>>(json) ?? new List<Transaction>();
        }
        else
        {
            Transactions = new List<Transaction>();
        }
    }

    public void Save()
    {
        string json = JsonSerializer.Serialize(Transactions);
        File.WriteAllText(_transactionsFile, json);
    }
}
### Шаг 4: Создайте главную страницу и ViewModel
Добавьте Page (HomePage.xaml) и соответствующую HomeViewModel. В XAML расположите Frame и кнопки навигации. В коде задайте начальную страницу.

### Шаг 5: Реализуйте добавление транзакции
Используйте RelayCommand. После добавления новой транзакции обновите коллекцию и сохраните данные через BudgetContext.

### 6.3 Навигация между страницами
В главном окне добавьте Frame с именем MainFrame. В ViewModel используйте сервис навигации:

csharp
public class NavigationService : INavigationService
{
    private readonly Frame _frame;
    public NavigationService(Frame frame) { _frame = frame; }
    public void NavigateTo(Page page) => _frame.Navigate(page);
}
